How notifications fire

  • One webhook per run, not per deviation — keeps Slack/Discord channels readable.
  • Fires only when the Differ writes ≥ 1 deviation against baseline.
  • min_severity per target: a target with min_severity=high only fires when at least one of the run's deviations is high or critical.
  • HMAC opt-in: when secret_env is set on a generic target, requests get X-FANGS-Signature: sha256=<hex>. Slack/Discord ignore secret_env — the URL is the secret in those models.
  • Retry policy: 5 attempts max, exponential backoff (1s × 2^(n-1) ±25% jitter). 4xx (non-408/429) is permanent; 5xx and network errors retry.
  • Every attempt persists an audit row in the notifications table — view with fangs notifier history -run <run_id>.
