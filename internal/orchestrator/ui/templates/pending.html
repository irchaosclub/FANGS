How the queue is built

  • A run appears here when the Differ writes at least one deviation row for it.
  • It leaves the queue when:
    • fangs baseline promote <run> accepts the run's fingerprint set into baseline (the deviation rows are cleared and the run is flagged is_baseline=true), or
    • The next run of the same package observes the same values and they enter baseline via auto-promote on a zero-deviation run (this only happens if the values were never novel again — usually the operator has to act first), or
    • The 90-day event retention prune removes the evidence events. Deviation rows themselves stay (they're findings, not raw evidence), but the "click for evidence" link breaks past the retention horizon.
  • The queue is sorted by max severity descending, then by most recent detection.
